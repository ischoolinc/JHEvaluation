## 目標

修改「匯入領域讀卡」判斷規則：

同一位學生可以在同一個匯入文字檔內，匯入不同領域的補考成績。

唯一值判斷由原本：

```text
學號
````

調整為：

```text
學號 + 領域名稱
```

也就是：

```text
同一學號 + 不同領域：允許匯入
同一學號 + 相同領域：視為重複資料，不允許匯入
```

其他匯入邏輯不更動。

修改完成後，請記錄在：

```text
高雄匯入補考讀卡成績調整0511.md
```

---

## 修改檔案

```text
ImportStartupForm.cs
```

---

## 修改原則

1. 只修改「同一檔案內重複資料判斷」。
    
2. 不調整文字檔解析邏輯。
    
3. 不調整領域代碼轉換邏輯。
    
4. 不調整學生查詢邏輯。
    
5. 不調整補考成績寫入邏輯。
    
6. 不調整 `JHSemesterScore.Update()` 更新流程。
    
7. 保持原本錯誤訊息流程與驗證流程。
    
8. 修改範圍要保守，避免影響其他匯入功能。
    

---

## 目前問題

目前程式在檢查同一檔案內是否重複資料時，只用學號判斷。

類似邏輯如下：

```csharp
List<string> s_numbers = new List<string>();

foreach (var dr in drCollection)
{
    if (!s_numbers.Contains(dr.StudentNumber))
    {
        s_numbers.Add(dr.StudentNumber);
    }
    else
    {
        msgList.Add("學號:"+dr.StudentNumber+"，資料重覆，請檢察資料來源txt是否有重覆填寫的學號資料。");
    }
}
```

這會造成以下資料被誤判為重複：

```text
學號 001，領域 語文
學號 001，領域 數學
```

實際需求是這兩筆應該允許匯入。

---

## 要修改成的規則

同一檔案內的唯一值改成：

```text
學號 + 領域名稱
```

允許：

```text
學號 001，領域 語文
學號 001，領域 數學
學號 001，領域 自然科學
```

不允許：

```text
學號 001，領域 語文
學號 001，領域 語文
```

---

## 修改步驟

### Step 1：找到重複學號檢查區塊

在 `ImportStartupForm.cs` 中搜尋：

```csharp
List<string> s_numbers
```

或搜尋錯誤訊息：

```csharp
資料重覆
```

找到目前檢查同一檔案內重複學號的邏輯。

---

### Step 2：將 `List<string> s_numbers` 改成 `HashSet<string>`

將原本：

```csharp
List<string> s_numbers = new List<string>();
```

改成：

```csharp
HashSet<string> studentDomainKeys = new HashSet<string>();
```

原因：

`HashSet<string>` 適合用來判斷唯一值，語意也比 `List.Contains()` 清楚。

---

### Step 3：將唯一值由「學號」改成「學號 + 領域」

將原本類似這段：

```csharp
if (!s_numbers.Contains(dr.StudentNumber))
{
    s_numbers.Add(dr.StudentNumber);
}
else
{
    msgList.Add("學號:"+dr.StudentNumber+"，資料重覆，請檢察資料來源txt是否有重覆填寫的學號資料。");
}
```

改成：

```csharp
// 同一檔案內，允許相同學號匯入不同領域。
// 唯一值改為：學號 + 領域名稱。
// 不允許同一學號 + 同一領域重複出現。
string studentNumber = dr.StudentNumber == null ? "" : dr.StudentNumber.Trim();
string domainName = dr.Domain == null ? "" : dr.Domain.Trim();

string studentDomainKey = studentNumber + "_" + domainName;

if (!studentDomainKeys.Contains(studentDomainKey))
{
    studentDomainKeys.Add(studentDomainKey);
}
else
{
    msgList.Add(
        "學號:" + studentNumber +
        "，領域:" + domainName +
        "，資料重覆，請檢查資料來源 txt 是否有重覆填寫相同學號與領域的資料。"
    );
}
```

## 注意事項

### 1. 不要改補考成績寫入邏輯

以下邏輯不要修改：

```csharp
jhssr.Domains[dr.Domain].ScoreMakeup = dr.Score;
```

原因：

這段本來就是依照不同領域寫入補考成績，已經符合需求。

---

### 2. 不要改領域判斷方式

以下類似邏輯不要修改：

```csharp
jhssr.Domains.ContainsKey(dr.Domain)
```

原因：

目前系統是用 `dr.Domain` 對應學期成績中的領域名稱，這次需求不是調整領域辨識方式。

---

### 3. 不要改學生查詢方式

以下類似邏輯不要修改：

```csharp
studentNumberToStudentIDs
```

原因：

學生仍然是透過學號對應學生 ID。

這次只調整「同一匯入檔案內是否重複」的判斷規則。

---

### 4. 不要改文字檔格式驗證

以下流程不要修改：

```csharp
ValidateTextFiles
RawDataCollection
DataRecordCollection
```

原因：

這次不是調整文字檔格式，也不是調整讀卡解析規則。

## 建議順手檢查

如果程式內有這個欄位：

```csharp
msg_DuplicatedDict
```

請確認每次匯入前是否有清空。

建議在 `_worker.DoWork` 開始處，若已經有：

```csharp
msgList.Clear();
```

可補上：

```csharp
msg_DuplicatedDict.Clear();
```

目的：

避免同一個畫面連續匯入時，上一輪的覆蓋警告訊息殘留。

注意：

這是安全性修正，不影響本次主要邏輯。

## 測試案例

### 測試 1：同一學生，不同領域

匯入資料：

```text
學號 001，領域 語文
學號 001，領域 數學
```

預期結果：

```text
允許匯入
```

---

### 測試 2：同一學生，相同領域

匯入資料：

```text
學號 001，領域 語文
學號 001，領域 語文
```

預期結果：

```text
顯示重複資料錯誤
不允許匯入
```

錯誤訊息應包含：

```text
學號:001，領域:語文，資料重覆
```

---

### 測試 3：不同學生，相同領域

匯入資料：

```text
學號 001，領域 語文
學號 002，領域 語文
```

預期結果：

```text
允許匯入
```

---

### 測試 4：同一學生，多個不同領域

匯入資料：

```text
學號 001，領域 語文
學號 001，領域 數學
學號 001，領域 自然科學
學號 001，領域 社會
```

預期結果：

```text
允許匯入
```

---

### 測試 5：同一學生不同領域，但其中一筆領域不存在

匯入資料：

```text
學號 001，領域 語文
學號 001，領域 不存在的領域
```

預期結果：

```text
語文可進入後續判斷
不存在的領域仍依照原本邏輯顯示「學生在該學年度學期並無該領域成績」
```

注意：

這次不修改領域不存在的判斷。

## 驗收條件

完成後請確認：

-  同一檔案內，相同學號可以匯入不同領域。
    
-  同一檔案內，相同學號 + 相同領域仍會被判斷為重複。
    
-  補考成績仍寫入 `jhssr.Domains[dr.Domain].ScoreMakeup`。
    
-  領域不存在時，仍維持原本錯誤處理。
    
-  學生不存在時，仍維持原本錯誤處理。
    
-  文字檔格式錯誤時，仍維持原本錯誤處理。
    
-  其他匯入邏輯不更動。
    
-  修改完成後，紀錄到 `高雄匯入補考讀卡成績調整0511.md`。

## 修改紀錄內容

請在 `高雄匯入補考讀卡成績調整0511.md` 記錄：

```md
# 高雄匯入補考讀卡成績調整 0511

## 調整目標

調整匯入領域讀卡補考成績的重複資料判斷規則。

原本同一檔案內只要學號重複，就會被判定為重複資料。

調整後改為以「學號 + 領域名稱」作為唯一值。

## 調整內容

1. 將同一檔案內重複判斷由「學號」改為「學號 + 領域名稱」。
2. 允許同一位學生在同一檔案內匯入不同領域的補考成績。
3. 同一位學生若同一領域重複出現，仍判定為重複資料。
4. 補考成績寫入邏輯不更動。
5. 領域辨識邏輯不更動。
6. 學生查詢與文字檔格式驗證邏輯不更動。

## 影響範圍

修改檔案：

- ImportStartupForm.cs

主要影響：

- 匯入補考讀卡成績時，同一學生不同領域可同時匯入。

不影響：

- 文字檔格式驗證
- 領域代碼轉換
- 學生查詢
- 學期領域成績查詢
- 補考成績寫入欄位
- JHSemesterScore.Update 更新流程

## 驗證結果

- 同一學生 + 不同領域：可匯入。
- 同一學生 + 相同領域：仍會顯示重複資料。
- 不同學生 + 相同領域：可匯入。
- 領域不存在：維持原本錯誤訊息。
```
