# 高雄匯入補考讀卡成績調整 0511

## 調整目標

調整匯入領域讀卡補考成績的重複資料判斷規則。

原本同一檔案內只要學號重複，就會被判定為重複資料。

調整後改為以「學號 + 領域名稱」作為唯一值。

## 調整內容

1. 將同一檔案內重複判斷由「學號」改為「學號 + 領域名稱」。
2. 允許同一位學生在同一檔案內匯入不同領域的補考成績。
3. 同一位學生若同一領域重複出現，仍判定為重複資料。
4. 補考成績寫入邏輯不更動。
5. 領域辨識邏輯不更動。
6. 學生查詢與文字檔格式驗證邏輯不更動。

## 影響範圍

修改檔案：

- ImportStartupForm.cs

主要影響：

- 匯入補考讀卡成績時，同一學生不同領域可同時匯入。
- 每次匯入開始時一併清空 `msg_DuplicatedDict`，避免連續匯入時上一輪覆蓋警告殘留。

不影響：

- 文字檔格式驗證
- 領域代碼轉換
- 學生查詢
- 學期領域成績查詢
- 補考成績寫入欄位
- JHSemesterScore.Update 更新流程

## 驗證結果

- 同一學生 + 不同領域：可匯入。
- 同一學生 + 相同領域：仍會顯示重複資料。
- 不同學生 + 相同領域：可匯入。
- 領域不存在：維持原本錯誤訊息。
