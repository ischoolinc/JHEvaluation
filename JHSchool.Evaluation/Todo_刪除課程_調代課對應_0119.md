## Todo 1 — 在 rbButton.Click 增加調代課筆數檢查與使用者確認
### 1.1 建立 helper：取得 dc_bind_key 筆數（依 courseIDList）
在 `Program.cs`（建議與 `DeleteCourseWithLogSQL` 同區塊）新增方法：

- 方法簽名建議：
  - `private static int GetDcBindKeyCountByCourseIDs(List<string> courseIDList)`
- 行為：
  - `courseIDList.Count == 0` → 回傳 `0`
  - 將 courseIDList 組成 `course_data_row`（沿用既有 UNION ALL 產生資料列的風格）
  - SQL：
    - 以 `dc_bind_key.ref_course_id` inner join `course_data_row.id`
    - `SELECT COUNT(*)`
  - 回傳整數筆數

> 實作細節要求：  
> - 使用 `QueryHelper` 查詢（與專案既有慣例一致）  
> - 避免把 SQL 寫死在 Click 事件內（保持可讀性）

### 1.2 在 rbButton.Click 中插入確認流程
在你既有 Click 事件中，**完成 `courseIDList` 組裝後**，插入：

- 呼叫 `GetDcBindKeyCountByCourseIDs(courseIDList)` 取得 `dcBindCount`
- 若 `dcBindCount > 0`：
  - 顯示確認視窗（MsgBox.Show Yes/No）
  - 文字建議：
    - 標題：`刪除調代課確認`
    - 內容：`發現所選課程中，尚有 {dcBindCount} 筆「調代課對應資料」。\n刪除課程將一併刪除調代課資料，是否繼續？`
  - 使用者選擇 No → `return;`

> 插入位置建議：  
> - 在成績檢查（sce_take / DeleteCourseWarningForm）之前，確保 UX 順序清楚：先確認調代課，再確認成績刪除。

## Todo 2 — 在 DeleteCourseWithLogSQL() 新增 dc_bind_key 刪除與 log
### 2.1 新增 CTE：delete_dc_bind_key
在 `WITH ...` CTE 串接中（緊接 `course_data_row` 後、`delete_course` 前）新增：

- CTE 名稱：`delete_dc_bind_key`
- 行為：
  - `DELETE FROM dc_bind_key`
  - 條件：`dc_bind_key.ref_course_id IN (SELECT id FROM course_data_row)`
  - `RETURNING dc_bind_key.*`

### 2.2 新增 CTE：insert_dc_bind_key_log（比照既有 log 寫法）
新增一段插入 `log` 的 CTE（或沿用你目前的 `insert_sce_take_log_student_data AS (...) INSERT INTO log ...` 的風格，擇一）：

- action_type：`Record`
- action：`課程_刪除`
- target_category：`course`
- target_id：使用 `delete_dc_bind_key.ref_course_id`
- action_by：`刪除_課程_調代課`
- description 建議（可微調，但要包含課程ID與使用者）：
  - `課程調代課資料刪除，課程ID「{ref_course_id}」，使用者「{actor}」`

> 注意：  
> - 你目前 SQL 最後是「一段 INSERT INTO log ... SELECT ...」直接接在 CTE 後面。  
> - 若要維持一致性，建議將 `insert_dc_bind_key_log` 做成 CTE，然後最後仍維持既有的「INSERT INTO log ...」寫法（或改成最後的主 INSERT 前置多一個 INSERT CTE，但要確保 SQL 可執行）。
