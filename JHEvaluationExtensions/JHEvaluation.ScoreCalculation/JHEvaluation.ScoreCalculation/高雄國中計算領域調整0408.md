# 高雄國中計算領域調整（2026-04-08）

## 修改項目

* **修正英語累加錯誤**：英語科目加權成績由 `engScore = +...` 改為 `engScore += ...`，避免多筆英語科目時僅保留最後一筆。
* **語文領域支援單科／任意組合**：`langScore` 在 `weight > 0` 時，依「國語文、英語、本土語文」有無之七種組合分別加權平均（與既有 `ScoreOrigin` 分支邏輯對齊）。
* **修正本土語文補考未納入問題**：進入「無補考則擇優」分支的條件，由僅檢查國語文、英語補考，改為一併檢查本土語文補考（`localMakeupScore`）。

## 影響範圍

* 僅 **`Program.Mode == ModuleMode.KaoHsiung`** 下「高雄專用語文領域成績結算」區塊內之 `CalculateDomainScore()` 行為。
* 其他縣市模式與非語文領域邏輯未變更。

## 修改檔案

* `BigFunction/SemesterScoreCalculator.cs`

## 測試結果

| 情境 | 預期 |
|------|------|
| 1. 只有國語文 | 語文領域存在；`langScore` = 國語文加權／總權重 |
| 2. 只有英語 | 語文領域存在；`langScore` = 英語加權／總權重；多筆英語時為加總後平均 |
| 3. 只有本土語文 | 語文領域存在；`langScore` = 本土語文加權／總權重 |
| 4. 國語文 + 英語 | 兩科加權和／總權重 |
| 5. 國語文 + 本土語文 | 兩科加權和／總權重 |
| 6. 英語 + 本土語文 | 兩科加權和／總權重 |
| 7. 國語文 + 英語 + 本土語文 | 三科加權和／總權重 |

**實際驗證**：請於高雄模式以實際學期資料跑分，對照上述七種科目組合，確認語文領域是否存在、`ScoreOrigin`／`Value`、英語多科目累加，以及僅本土語文有補考時是否走補考擇優流程。
