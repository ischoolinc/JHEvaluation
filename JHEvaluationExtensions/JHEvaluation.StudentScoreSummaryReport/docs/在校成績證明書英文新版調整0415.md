# 在校成績證明書（英文新版）調整 — 2026-04-15

## 1. 調整目標

將一般領域（尤其語文）可顯示的科目數上限由 **6** 調整為 **10**，使：

- DataTable 欄位可建立到第 10 科
- 成績 / 等第 / 權數（含原始、平均）可正確寫入
- Word MailMerge 能對應到新增欄位
- 不影響彈性課程既有 7~18 的欄位與邏輯

## 2. 修改內容

**檔案：** `PrintForm_StudentScoreCertificattion_English.cs`

- **新增常數**：`NormalDomainSubjectMax = 10`
- **DataTable 欄位建立（科目）**：
  - 將一般領域的 `科目名稱/權數/成績/等第/原始/平均` 欄位建立由 1~6 擴為 1~10
  - 以 `domain == "彈性課程" ? 6 : NormalDomainSubjectMax` 控制，確保彈性課程仍維持原本 1~6 + 另段 7~18 的建立方式
- **subjectScoreType_list / subjectLevelType_list / subjectCredit_list**：
  - 同樣以 `domain == "彈性課程" ? 6 : NormalDomainSubjectMax` 生成 key 前綴，讓一般領域可覆蓋到科目 10
- **科目數上限判斷**：
  - `if (subjectCourseCount > 6)` 改為 `if (subjectCourseCount > NormalDomainSubjectMax)`

## 3. Word 樣板同步（必要）

需於英文版 Word 樣板新增（以語文為例）：

- `語文_科目名稱7` ~ `語文_科目名稱10`
- `語文_科目7` ~ `語文_科目10` 的：
  - `權數1~6`
  - `成績1~6`
  - `等第1~6`
  - `原始成績1~6`
  - `原始等第1~6`
  - `平均成績`
  - `平均成績等第`

## 4. 測試建議

- 語文科目 6 科：行為應與原本一致
- 語文科目 7~10 科：必須全部顯示
- 語文科目 > 10：顯示前 10，並出現原本的超量提示
- 其他一般領域（如數學）：亦可顯示到 10，不影響既有資料

## 保守安全修正：避免 DataTable 欄位不存在例外

- 問題：執行時出現「資料行 '領域_彈性學習課程_平均成績' 不屬於資料表」
- 原因：實際資料產生的 key 與 DataTable 已建立欄位名稱不一致
- 修正：在填入 `row[key]` 前，先加上 `table.Columns.Contains(key)` 判斷
- 影響範圍：
  - 領域分數
  - 領域等第
  - 領域權數
  - 科目分數
  - 科目等第
  - 科目權數
- 原則：保守修正，不更動原本 `DomainList` 與領域分類邏輯
