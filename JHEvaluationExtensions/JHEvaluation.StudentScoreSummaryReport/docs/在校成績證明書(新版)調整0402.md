# 在校成績證明書(新版)調整 — 2026-04-02

## 1. 問題說明

`DataTable` 已建立彈性課程第 7~18 個科目的合併欄位，但建立 `subjectScore_dict`、`subjectLevel_dict`、`subjectCredit_dict` 時所用的 `subjectScoreType_list`、`subjectLevelType_list`、`subjectCredit_list` 只涵蓋各領域科目 1~6（含彈性課程 1~6），未納入 **彈性課程_科目7~18** 的 key 前綴。後續迴圈會為每個前綴加上學期後綴 `1`~`6`，因此缺少 7~18 的前綴時，第 7~18 個彈性科目的成績、等第、權數、平均無法寫入對應欄位。

## 2. 修改內容

**檔案：** `PrintForm_StudentScoreCertificattion.cs`

在下列三處，於既有 `foreach (string domain in DomainList)` 迴圈之後，各補上一段 `for (int a = 7; a <= 18; a++)`，僅針對 **彈性課程** 加入：

| 區塊 | 補齊的 key 前綴 |
|------|----------------|
| `subjectScoreType_list` | `彈性課程_科目{a}_成績`、`原始成績`、`平均成績` |
| `subjectLevelType_list` | `彈性課程_科目{a}_等第`、`原始等第`、`平均成績等第` |
| `subjectCredit_list` | `彈性課程_科目{a}_權數` |

未更動：一般領域科目 1~6、`DataTable` 欄位命名、`AlternativeCourse` 與超過 18 筆之限制與提示。

## 3. 測試結果

- 未於此環境執行實際報表；邏輯上字典會與既有 `彈性課程_科目1~6` 相同方式產生 `…_成績1`~`…_成績6` 等鍵，第 7~18 個彈性科目應可寫入。
- 建議依 Todo 所列案例（6、7、12、18、超過 18 個彈性課程）於系統內驗證 Word 合併輸出。
