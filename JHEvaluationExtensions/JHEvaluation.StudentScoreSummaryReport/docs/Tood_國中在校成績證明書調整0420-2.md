# 二、關鍵問題說明（現況）
目前 Global.cs：

- 英文版：10 科
- 中文版：6 科 ❌

原因：

```csharp
int domainSubjectMax = (doc == "英文版" && domain != "彈性課程") 
    ? subjectMax 
    : DefaultDomainSubjectMax;
````

👉 中文版會走 `DefaultDomainSubjectMax = 6`

## 核心原則

👉 一般領域統一使用 10 科（中英文共用）
👉 彈性課程固定 18 科

# 四、Step-by-Step 修改

---

## Step 1：新增共用常數（最重要）

【檔案】
Global.cs

【搜尋】
EnglishDomainSubjectMax / DefaultDomainSubjectMax

【新增或調整】

```csharp
const int NormalDomainSubjectMax = 10;   // 一般領域統一 10 科
const int FlexibleDomainSubjectMax = 18; // 彈性課程 18 科
```

【注意】

1. 若 `EnglishDomainSubjectMax` 只用於此，可移除或保留（不影響）
2. `DefaultDomainSubjectMax = 6` 可保留（避免影響其他模組）

---

## Step 2：統一 subjectMax

【搜尋】

```csharp
int subjectMax = DefaultDomainSubjectMax;
```

以及：

```csharp
if (doc == "英文版")
    subjectMax = EnglishDomainSubjectMax;
```

【修改為】

```csharp
int subjectMax = NormalDomainSubjectMax;
```

【目的】
👉 不再依 doc（中文版/英文版）切換
👉 中英文都使用 10 科

---

## Step 3：修改領域科目數判斷（核心）

【搜尋關鍵字】

```csharp
doc == "英文版"
```

找到這段（通常出現 2 次以上）：

```csharp
int domainSubjectMax = (doc == "英文版" && domain != "彈性課程") 
    ? subjectMax 
    : DefaultDomainSubjectMax;
```

【修改為】

```csharp
int domainSubjectMax = (domain == "彈性課程") 
    ? FlexibleDomainSubjectMax 
    : subjectMax;
```

---

## Step 4：確認所有迴圈都用 domainSubjectMax

確認以下類型全部改為：

```csharp

```

## Step 5：修正「科目數上限判斷」

【搜尋】

```csharp

```
