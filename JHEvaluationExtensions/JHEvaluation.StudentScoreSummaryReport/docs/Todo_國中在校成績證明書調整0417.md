
# 任務名稱

修正 PrintForm_StudentScoreCertificattion_English.cs 原始成績讀取來源

# 目標

將「_原始成績」與「_原始等第」的資料來源
由 `subjectscore.Value.Score` 改為 `subjectscore.Value.ScoreOrigin`
並加入空值防護，避免無資料時發生錯誤或顯示異常。

---

# 修改檔案

PrintForm_StudentScoreCertificattion_English.cs

---

# 修改原則（非常重要）

1. ❗ 不可修改「_成績」邏輯
2. ❗ 只修改「_原始成績」與「_原始等第」
3. ❗ 不影響其他欄位或資料流
4. ❗ 採用保守寫法（避免 null crash）
5. ❗ 欄位無值時統一填入空字串 ""

---

# 修改內容

## 一、修改 _原始成績

### 原本（錯誤寫法）

```csharp
subjectscore.Value.Score
```

### 修改為（安全寫法）

```csharp
subjectscore.Value.ScoreOrigin.HasValue 
    ? subjectscore.Value.ScoreOrigin.Value 
    : (decimal?)null
```

如果該欄位是 string：

```csharp
subjectscore.Value.ScoreOrigin.HasValue 
    ? subjectscore.Value.ScoreOrigin.Value.ToString() 
    : ""
```

---

## 二、修改 _原始等第

### 原本（錯誤寫法）

```csharp
_ScoreMappingConfig.ParseScoreEngName(subjectscore.Value.Score)
```

### 修改為（安全寫法）

```csharp
subjectscore.Value.ScoreOrigin.HasValue 
    ? _ScoreMappingConfig.ParseScoreEngName(subjectscore.Value.ScoreOrigin.Value) 
    : ""
```

---

## 三、需要修改的區塊（重點搜尋）

請搜尋以下關鍵字逐一修正：

### 1️⃣ _原始成績

```
_原始成績
```

### 2️⃣ _原始等第

```
_原始等第
```

---

# 注意事項（關鍵）

1. ⚠️ ScoreOrigin 是 Nullable<decimal>
   → 一定要加 `.HasValue`

2. ⚠️ 不可直接：

```csharp
subjectscore.Value.ScoreOrigin.Value
```

（會噴錯）

3. ⚠️ DataTable 欄位如果是 string
   → 一律用 "" 不要用 null

4. ⚠️ 不要修改：

* _成績
* 等第（非原始）
* Domain 邏輯
* 彈性課程判斷

---

# 驗證方式

修改後請確認：

1. 有原始成績資料：

   * _成績 ≠ _原始成績（正常分離）

2. 無原始成績：

   * _原始成績 = ""
   * _原始等第 = ""

3. 報表可正常產出（無例外）
